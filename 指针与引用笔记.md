# 指针与引用笔记

## 指针（pointer）`*`

指针存的是某块内存的**地址**，不是值本身。

```cpp
int x = 42;
int *p = &x;   // p 存的是 x 的地址，比如 0x7fff1234
*p = 100;      // 通过地址修改 x（解引用）
p = nullptr;   // p 可以改变指向，甚至指向空
```

### 为什么用指针而不直接传值？

GPU 处理的数据（token 矩阵、send buffer 等）可能有几个 GB，不可能把整块数据"传进"函数，所以只传一个地址，函数通过地址去访问数据。

### 指针算术

指针可以做加减运算，偏移到数组的某个元素：

```cpp
std::byte *token_ptr = send_buffer + route.position * token_stride;
//                     ^ 基地址      ^ 偏移量
```

### `**` 双重指针

`**sync_ptrs` 是指针的指针——存的是一个地址，那个地址处存的还是另一个地址，相当于一个二维数组或指针数组。

---

## 引用（reference）`&`

引用是变量的**别名**，不是独立变量，本身不占独立空间。

```cpp
int x = 42;
int &r = x;    // r 是 x 的别名
r = 100;       // 直接就是修改 x，不需要 *
// r = nullptr; ← 不合法！引用不能重新绑定，也不能为空
```

---

## 对比表

| | 指针 `*` | 引用 `&` |
|---|---|---|
| 能否为空 | 能（`nullptr`） | 不能 |
| 能否改变指向 | 能 | 不能，绑定后永久 |
| 使用时需要解引用 | 需要写 `*p` | 不需要，直接用 `r` |
| 能否做算术偏移 | 能 | 不能 |
| 底层实现 | 存地址的变量 | 编译器通常也用指针实现，只是隐藏了细节 |

---

## 这个文件里为什么用指针不用引用？

来自 `a2a_dispatch_send.cu`，原因有两个：

1. **参数可能为空**：如 `bound_m_ptr` 需要判断 `nullptr`，引用不能为空
   ```cpp
   const size_t num_send_tokens = bound_m_ptr ? *bound_m_ptr : num_tokens;
   ```

2. **需要指针算术**：如计算 token 在 buffer 中的位置，引用不支持
   ```cpp
   std::byte *token_ptr = send_buffer + route.position * token_stride;
   ```
