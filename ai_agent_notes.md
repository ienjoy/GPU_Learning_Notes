## 什么是 Agent（智能体）？

**一句话定义：**
> Agent = 能自主决策、调用工具、完成多步骤任务的 AI 程序

普通 AI（如 ChatGPT 单次对话）：你问 → 它答 → 结束  
Agent：你给目标 → 它自己拆解步骤 → 调用工具 → 执行 → 反思 → 完成目标

---

## Agent 的基本结构

```
┌─────────────────────────────────────┐
│              Agent 核心             │
│                                     │
│  🧠 LLM（大脑）                     │
│     ↓ 决策：下一步做什么？           │
│  🔧 工具调用（Tools）               │
│     - 搜索网页                      │
│     - 读写文件                      │
│     - 执行代码                      │
│     - 调用 API                      │
│     ↓                               │
│  📋 记忆（Memory）                  │
│     - 短期：当前对话上下文           │
│     - 长期：数据库/向量检索          │
│     ↓                               │
│  🔄 循环：执行 → 观察结果 → 再决策  │
└─────────────────────────────────────┘
```

---

## Agent 的核心能力

| 能力 | 说明 | 例子 |
|---|---|---|
| **规划（Planning）** | 把大目标拆成小步骤 | "帮我写报告" → 搜索→整理→写作→校对 |
| **工具使用（Tool Use）** | 调用外部工具 | 搜索、代码执行、邮件发送 |
| **记忆（Memory）** | 记住上下文和历史 | 记住你的偏好、之前的对话 |
| **反思（Reflection）** | 检查自己的输出 | 发现错误 → 重试 |
| **多 Agent 协作** | 多个 Agent 分工 | 一个搜索、一个写作、一个审核 |

## 主流 Agent 框架

### 🐍 Python 生态

| 框架 | 特点 | 适合场景 |
|---|---|---|
| **LangChain** | 最成熟，工具链丰富 | 通用 Agent，RAG 检索 |
| **LangGraph** | 基于图结构的流程控制 | 复杂多步骤工作流 |
| **CrewAI** | 多 Agent 角色扮演协作 | 团队分工型任务 |
| **AutoGen** | 微软出品，多 Agent 对话 | 代码生成、自动调试 |

### 🖥️ 无代码 / 低代码

| 工具 | 特点 | 适合场景 |
|---|---|---|
| **Dify** | 可视化界面，开源可自部署 | 快速搭建 AI 应用 |
| **n8n** | 自动化工作流，连接各种 API | 业务流程自动化 |

## Agent 能做什么？（实际案例）

### 研究助手 Agent
```
目标：调研"GPU 显存优化"技术
→ 搜索论文
→ 读取 PDF 摘要
→ 对比各方法
→ 输出总结报告
```

### 代码 Agent（如 GitHub Copilot）
```
目标：修复这个 bug
→ 读代码文件
→ 定位问题
→ 修改代码
→ 运行测试
→ 验证通过
```

### 个人助理 Agent
```
目标：帮我安排明天的日程
→ 读邮件
→ 分析优先级
→ 写入日历
→ 发送提醒
```

## Agent vs 普通 AI 对比

| 特性 | 普通 AI（ChatGPT 问答） | Agent |
|---|---|---|
| 执行步骤 | 单步 | 多步循环 |
| 使用工具 | ❌ | ✅ |
| 自主决策 | ❌ | ✅ |
| 需要人介入 | 每步都需要 | 目标级别才需要 |
| 典型用时 | 秒级 | 分钟到小时 |

## 怎么开始学 Agent？

### 入门路线
1. **先理解 LLM 基础** → 会用 ChatGPT API
2. **学 LangChain 基础** → 工具调用、链式调用
3. **跑一个简单 Agent** → ReAct 模式（推理+行动）
4. **多 Agent 协作** → 用 CrewAI 或 LangGraph

### 最快上手：用 Dify 无代码体验
1. 去 [dify.ai](https://dify.ai) 注册
2. 创建一个 Agent 应用
3. 给它配置工具（搜索、代码执行）
4. 测试：让它帮你做一个多步骤任务

### 推荐资源
- **LangChain 官方文档**：python.langchain.com
- **CrewAI GitHub**：github.com/joaomdmoura/crewAI
- **吴恩达 AI Agents 课程**（DeepLearning.AI，免费）

## 和 GPU 学习的关联

Agent 框架本身是 Python 应用层，但：
- **推理加速**：大模型推理跑在 GPU 上（CUDA kernel）
- **Batch 推理**：多 Agent 并发请求 → GPU 并行处理
- **本地部署**：在自己的 GPU 上跑 LLaMA + Agent 框架

> 学了 CUDA 之后，你可以理解 Agent 底层的推理引擎（vLLM、TensorRT-LLM）是怎么用 GPU 加速的。
